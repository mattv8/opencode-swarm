# v7.18.0 - Standard Parallelization Config Wiring

## Summary

Standard non-Lean parallelization is now wired into architect dispatch guidance and runtime QA barriers. Users can enable the `parallelization` block in `opencode-swarm.json` to authorize bounded fan-out for independent coder tasks and all applicable Stage B gate groups without enabling Lean Turbo.

## Behavior

- `parallelization.enabled: true` injects explicit standard parallelization guidance into the architect prompt.
- The delegation gate resolves effective standard parallelization settings at runtime.
- `maxConcurrentTasks` clamps both coder fan-out and Stage B gate-group fan-out.
- `max_coders` and `max_reviewers` remain per-role caps, bounded by `maxConcurrentTasks`.
- When Stage B parallelization is enabled, the architect dispatches every applicable Stage B gate together for a task: reviewer, verification test_engineer, and conditional adversarial test_engineer when required.
- The delegation gate records adversarial test_engineer as a distinct `adversarial_test_engineer` Stage B marker so reviewer plus verification test_engineer cannot prematurely satisfy a task whose adversarial gate was dispatched.
- Evidence files with an empty `required_gates` array no longer count as satisfying QA gates; at least one declared required gate must have matching evidence before evidence alone can pass completion.
- Lean Turbo remains separate; enabling standard parallelization does not require `/swarm turbo lean` and does not call Lean Turbo tools.
- Locked plan `execution_profile` values remain bounded by the global config; runtime guardrails use the stricter bound, and an explicit global `parallelization.enabled: false` remains a hard disable.
- Global config uses `$XDG_CONFIG_HOME/opencode/opencode-swarm.json` or `<home>/.config/opencode/opencode-swarm.json` on Linux, macOS, and native Windows. Project config remains `<project>/.opencode/opencode-swarm.json` and deep-merges over global config.
